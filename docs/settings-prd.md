# ProjectHub Dashboard Settings - Product Requirements Document

## 1. Обзор раздела настроек

### 1.1 Цель
Создать централизованный раздел настроек, позволяющий каждому пользователю полностью персонализировать дашбоард под свои потребности, рабочий процесс и предпочтения.

### 1.2 Целевая аудитория
- Разработчики с разными IDE (VS Code, Windsurf, Antigravity, JetBrains и др.)
- Команды с разными предпочтениями локализации
- Пользователи с разными паттернами работы с проектами

## 2. Модули настроек

### 2.1 Общие настройки (General Settings)

#### Язык интерфейса (Interface Language)
- **Поддерживаемые языки:**
  - 🇷🇺 Русский (Russian) - default
  - 🇬🇧 English (English)
  - 🇨🇳 中文 (Chinese Simplified)
  - 🇯🇵 日本語 (Japanese) - future
  - 🇩🇪 Deutsch (German) - future

- **Функционал:**
  - Переключатель языка с предпросмотром
  - Автоопределение системной локали при первом запуске
  - Кеширование переводов для производительности
  - Fallback на английский при отсутствии перевода

#### Тема оформления (Theme)
- **Варианты:**
  - GitHub Dark (default)
  - GitHub Light
  - Midnight OLED (AMOLED-friendly)
  - High Contrast (для доступности)
  - Custom (пользовательские цвета)

- **Автоматические режимы:**
  - Следовать системной теме (dark/light)
  - Переключение по времени суток

### 2.2 Настройки редакторов кода (Editor Settings)

#### Конфигурация редакторов
**Структура записи редактора:**
```typescript
interface EditorConfig {
  id: string;              // Уникальный идентификатор (windsurf, vscode, cursor)
  name: string;            // Отображаемое имя
  icon: string;            // Путь к иконке или base64
  command: string;         // Команда запуска (windsurf, code, cursor)
  args_template: string;   // Шаблон аргументов: "{path}" или "-r {path}"
  color: string;           // Цвет кнопки в hex (#1f6feb)
  is_default: boolean;     // Редактор по умолчанию
  is_enabled: boolean;     // Включен в интерфейсе
  category: 'primary' | 'secondary' | 'custom';
}
```

**Предустановленные редакторы:**
1. **Windsurf** (Codeium)
   - Команда: `windsurf`
   - Аргументы: `{path}`
   - Цвет: #10b981 (emerald)

2. **VS Code**
   - Команда: `code`
   - Аргументы: `--folder-uri {path}`
   - Цвет: #0078d4 (blue)

3. **VS Code Insiders**
   - Команда: `code-insiders`
   - Цвет: #0078d4

4. **Cursor**
   - Команда: `cursor`
   - Цвет: #f97316 (orange)

5. **Antigravity** (Codeium)
   - Команда: `antigravity`
   - Аргументы: `{path}`
   - Цвет: #8b5cf6 (violet)

6. **JetBrains Fleet**
   - Команда: `fleet`
   - Цвет: #6366f1

7. **Zed**
   - Команда: `zed`
   - Цвет: #d946ef

8. **Terminal + Tmux (для хардкорщиков)**
   - Команда: `gnome-terminal`
   - Аргументы: `--working-directory={path}`
   - Цвет: #000000

**Функционал управления:**
- Drag-and-drop для сортировки приоритета
- Включение/выключение видимости в карточках
- Загрузка кастомной иконки (PNG/SVG, max 32x32)
- Тестовый запуск для проверки конфигурации
- Детекция установленных редакторов в системе

#### Расширенные настройки запуска
- **Параллельный запуск:** Открывать несколько проектов одновременно
- **New Window:** Форсировать открытие в новом окне
- **Wait/No-wait:** Ждать закрытия или отпустить сразу
- **Pre-launch script:** Команда перед открытием (например, git pull)
- **Post-launch script:** Команда после открытия

### 2.3 Настройки проектов (Project Settings)

#### Автосканирование
- **Интервал:** От 5 минут до 24 часов
- **Директории для сканирования:**
  - Добавление кастомных путей
  - Исключение папок (node_modules, .git, venv)
  - Глубина сканирования (1-5 уровней)

#### Правила автоопределения типа
- Включение/выключение детекторов
- Приоритет языков
- Кастомные паттерны файлов для определения типа

#### Шаблоны проектов
- Создание шаблонов для быстрого старта
- Дефолтные структуры папок
- Pre-commit hooks настройки

### 2.4 Настройки интерфейса дашбоарда (Dashboard UI)

#### Сетка проектов
- **Количество колонок:** 1-6 (auto по ширине экрана)
- **Размер карточки:** Compact / Normal / Large
- **Отображение в карточке:**
  - [x] Название проекта
  - [x] Путь
  - [x] Тип проекта
  - [x] Теги
  - [x] Git статус
  - [x] Docker статус
  - [x] Последнее открытие
  - [x] Счетчик открытий
  - [x] Кнопки быстрых действий

#### Боковая панель
- Порядок разделов (drag-and-drop)
- Свернуть/развернуть по умолчанию
- Кастомные категории (пользовательские группы)

#### Системный мониторинг
- Выбор отображаемых метрик (CPU/RAM/Disk/Network/GPU)
- Интервал обновления (5s - 5min)
- Пороговые значения для предупреждений
- История метрик (графики)

#### Шапка (Header)
- Компактный / Расширенный режим
- Показывать/скрыть статистику проектов
- Показывать/скрыть системные метрики
- Кастомное приветствие ("Доброе утро, {username}")

### 2.5 Настройки уведомлений (Notifications)

#### Типы уведомлений
- [ ] Новые проекты найдены
- [ ] Долгая активность без коммитов
- [ ] Docker контейнер упал
- [ ] Высокая нагрузка на систему
- [ ] Обновления зависимостей доступны
- [ ] Дневная сводка

#### Каналы доставки
- Browser notifications (Web Push)
- Telegram бот
- Email дайджест (ежедневный/еженедельный)
- Desktop notifications (Linux notify-send)

#### Тихие часы
- Не беспокоить: 22:00 - 08:00
- Фокус-режим: отключить все уведомления

### 2.6 Настройки Git интеграции

#### Автоматические действия
- [ ] Автоматический git status при открытии проекта
- [ ] Уведомление о незакоммиченных изменениях
- [ ] Авто-пулл при открытии
- [ ] Авто-коммит по расписанию (daily backup)

#### Визуализация
- Цветовая схема статусов веток
- Показывать количество коммитов ahead/behind
- Отображать автора последнего коммита

### 2.7 Настройки Docker интеграции

#### Мониторинг
- Автоопределение docker-compose файлов
- Отслеживание статуса контейнеров
- Автоподнятие контейнеров при открытии проекта
- Логи контейнеров в реальном времени

### 2.8 Настройки безопасности

#### Доступ к проектам
- Блокировка дашбоарда паролем/PIN
- Тайм-аут блокировки (1/5/15/30 минут)
- Шифрование заметок
- Безопасное удаление истории

#### API и интеграции
- Создание API ключей
- Webhook настройки
- CORS политики

### 2.9 Экспорт/Импорт и бэкап

#### Экспорт данных
- JSON dump всех настроек
- CSV экспорт списка проектов
- Архив с заметками и метаданными

#### Импорт
- Импорт из других тулов (Project Manager, Project Dashboard)
- Массовый импорт проектов из CSV

#### Синхронизация
- Синхронизация настроек через облако (GitHub Gist, Dropbox)
- Резервное копирование по расписанию
- Версионирование настроек

### 2.10 Клавиатурные сокращения (Hotkeys)

#### Глобальные шорткаты
- `Cmd/Ctrl + Shift + P` - Поиск проекта
- `Cmd/Ctrl + Shift + N` - Создать новый проект
- `Cmd/Ctrl + ,` - Открыть настройки
- `Cmd/Ctrl + R` - Обновить дашбоард
- `Esc` - Закрыть модальное окно

#### Навигация
- `J/K` - Следующий/предыдущий проект
- `Enter` - Открыть проект
- `F` - Открыть папку
- `1-9` - Запуск в редакторе #N

#### Кастомизация
- Возможность переопределить любой шорткат
- Конфликт-детекция
- Экспорт/импорт keymap

## 3. Техническая реализация

### 3.1 Структура БД

```sql
-- Таблица настроек пользователя
CREATE TABLE user_settings (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT DEFAULT 'default',
    key TEXT NOT NULL,
    value TEXT,
    value_type TEXT DEFAULT 'string', -- string, number, boolean, json
    category TEXT, -- general, editor, ui, etc.
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, key)
);

-- Таблица конфигураций редакторов
CREATE TABLE editor_configs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT DEFAULT 'default',
    editor_id TEXT NOT NULL,
    name TEXT NOT NULL,
    command TEXT NOT NULL,
    args_template TEXT DEFAULT '{path}',
    icon_path TEXT,
    color TEXT DEFAULT '#1f6feb',
    sort_order INTEGER DEFAULT 0,
    is_enabled BOOLEAN DEFAULT 1,
    is_default BOOLEAN DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, editor_id)
);

-- Таблица пользовательских категорий
CREATE TABLE user_categories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT DEFAULT 'default',
    name TEXT NOT NULL,
    icon TEXT,
    color TEXT,
    sort_order INTEGER DEFAULT 0
);

-- Локализация
CREATE TABLE translations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    lang_code TEXT NOT NULL, -- 'ru', 'en', 'zh'
    key TEXT NOT NULL,
    value TEXT NOT NULL,
    context TEXT, -- для контекста перевода
    UNIQUE(lang_code, key)
);
```

### 3.2 API Endpoints

```
GET    /api/settings                    # Получить все настройки
PUT    /api/settings                    # Сохранить настройки
GET    /api/settings/editors            # Список редакторов
POST   /api/settings/editors            # Добавить редактор
PUT    /api/settings/editors/{id}       # Обновить редактор
DELETE /api/settings/editors/{id}       # Удалить редактор
POST   /api/settings/editors/reorder    # Изменить порядок
POST   /api/settings/test-editor        # Тестовый запуск
GET    /api/settings/i18n/{lang}        # Получить переводы
GET    /api/settings/export             # Экспорт всех настроек
POST   /api/settings/import             # Импорт настроек
POST   /api/settings/reset              # Сброс к дефолтным
```
